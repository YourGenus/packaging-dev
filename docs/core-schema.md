# Genus Core Schema v0.1  
Foundational design architecture for genres, product categories, types, parts, and parameters.  
---

## 1. Purpose  
This schema defines the core building blocks of the Genus design system. It provides a unified structure for:

- Design guidelines  
- Product architecture  
- Variant generation  
- Future supply‑chain and operations data  
- Automated product configuration  

This document is versioned and will evolve as the platform grows.

---

## 2. Genres  
Identities that influence color, tone, and emotional expression across the Genus ecosystem.

Each genre includes:  
- **id**: unique key  
- **name**: human-readable name  
- **tags**: emotional or functional associations  
- **colors**: primary, secondary, accent  

### 2.1 Genre Schema  
```json
{
  "id": "string",
  "name": "string",
  "tags": ["string"],
  "colors": {
    "primary": "hex",
    "secondary": "hex",
    "accent": "hex"
  }
}
```

### 2.2 Current Genres  
(Directly from the JSON source)

- Citizen  
- Athlete  
- Dreamer  
- Metropolitan  
- Traveler

## 4. Types  
Form factors that define physical dimensions, volume, and required parts.

Each type includes:  
- **id**: unique key  
- **name**: type name  
- **description**: purpose or usage context  
- **parameters**: measurable attributes (volume, height, diameter)  
- **parts**: list of required components  

### 4.1 Type Schema  
```json
{
  "id": "string",
  "name": "string",
  "description": "string",
  "parameters": {
    "volume": "number (optional)",
    "height": "number",
    "diameter": "number"
  },
  "parts": ["string"]
}
```

### 4.2 Current Types  
- Standard  
- Extended  
- Compact  
- Insert Only  

---

## 5. Parts  
Physical components used across types.  
Parts are modular and reusable across product types.

Each part includes:  
- **id**: unique key  
- **name**: part name  
- **material**: e.g., PCR, PP, PET  
- **finish**: matte, gloss, soft-touch  
- **colorMode**:  
  - `"genre"` → inherits from genre colors  
  - `"fixed"` → uses a defined color  
- **dimensions** (optional): height, diameter, wall thickness  

### 5.1 Part Schema  
```json
{
  "id": "string",
  "name": "string",
  "material": "string",
  "finish": "string",
  "colorMode": "genre | fixed",
  "color": "hex (optional)",
  "dimensions": {
    "height": "number (optional)",
    "diameter": "number (optional)",
    "thickness": "number (optional)"
  }
}
```

### 5.2 Example Parts  
```json
{
  "parts": [
    {
      "id": "shell",
      "name": "Outer Shell",
      "material": "ABS",
      "finish": "matte",
      "colorMode": "genre"
    },
    {
      "id": "insert",
      "name": "PCR Insert",
      "material": "PCR",
      "finish": "semi-gloss",
      "colorMode": "fixed",
      "color": "#F5F5F5"
    },
    {
      "id": "cap",
      "name": "Cap",
      "material": "PP",
      "finish": "matte",
      "colorMode": "genre"
    }
  ]
}
```

## 6. Parameters  
Quantitative attributes used for product configuration, manufacturing, and variant generation.

### 6.1 Parameter Types  
- **volume** (ml)  
- **height** (mm)  
- **diameter** (mm)  
- **weight** (g)  
- **threadType** (e.g., 24/410)  
- **compatibility** (list of compatible parts or types)  

### 6.2 Parameter Schema  
```json
{
  "name": "string",
  "unit": "string",
  "value": "number",
  "tolerance": "number (optional)"
}
```

## 7. Product Definition  
A product is generated by combining:  
- **genre**  
- **productCategory**  
- **type**  
- **parts**  
- **parameters**  

This enables automated variant generation and future supply-chain integration.

### 7.1 Product Schema  
```json
{
  "id": "string",
  "genre": "genre.id",
  "category": "productCategory.id",
  "type": "type.id",
  "parts": ["part.id"],
  "parameters": {
    "volume": "number",
    "height": "number",
    "diameter": "number"
  }
}
```

## 8. Variant Generation Rules  
The system can automatically generate product variants using:

### 8.1 Genre-Based Rules  
- Shell and cap inherit **genre colors**  
- Accent color may apply to:  
  - logo  
  - ring detail  
  - cap band  

### 8.2 Type-Based Rules  
- Dimensions and volume come from the type  
- Parts required by the type must be present  

### 8.3 Category-Based Rules  
- Cleanser → pump or flip-cap  
- Mist → spray actuator  
- Serum → dropper or precision pump  

### 8.4 Compatibility Rules  
- Insert must match shell diameter  
- Cap must match thread type  
- Volume must not exceed insert capacity
